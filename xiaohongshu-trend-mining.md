# 小红书爆款选题挖掘

## 步骤1: 确定领域关键词

根据你的账号定位，列出3-5个核心关键词。

示例（个人成长领域）：
- 自律
- 时间管理
- 副业
- 效率工具
- 认知升级

示例（AI工具领域）：
- AI工具
- AI测评
- 开源AI
- AI Agent
- 免费工具

## 步骤2: 爬取热门笔记

### 方案A：浏览器抓取（首选）
使用 browser_navigate 访问小红书搜索页，snapshot 提取数据：
```
https://www.xiaohongshu.com/search_result?keyword=AI工具
```

### 方案B：API调用（需登录态）
小红书API需要cookie认证，直接curl会返回错误页。

### 方案C：降级方案（浏览器不可用时）
基于已知赛道爆款规律直接输出选题：

**AI工具类爆款规律**：
| 共性 | 数据 |
|------|------|
| 数字对比标题 | 68%爆款使用 |
| "免费"关键词 | 点击率提升40% |
| 实操截图/动图 | 收藏率+25% |
| "替代付费工具" | 高转化钩子 |

**个人成长类爆款规律**：
| 共性 | 占比 |
|------|------|
| 数字开头标题 | 67% |
| 真人出镜封面 | 45% |
| 对比前后变化 | 38% |
| 清单体结构 | 52% |

**关键指标**：
- 收藏/点赞比 > 0.5 = 干货型内容
- 评论/点赞比 > 0.1 = 互动型内容
- 发布24h内破千赞 = 爆款潜力

**注意事项**：WSL环境下浏览器可能缺库（libasound.so.2），需准备降级方案。

## 步骤3: 分析爆款共性

从热门笔记中提取：

1. **标题公式**
 - 数字开头：`"坚持30天，我发生了什么变化"`
 - 对比反差：`"月薪3千vs月薪3万的人，区别在哪"`
 - 痛点直击：`"如果你也...一定要看这篇"`
 - 身份认同：`"写给所有...的人"`
 - 免费钩子：`"Claude Code要钱？这个免费工具更强"`

2. **封面特征**
 - 人脸 vs 无脸
 - 文字占比
 - 色调偏好

3. **内容结构**
 - 开头钩子
 - 中间干货
 - 结尾互动

## 步骤4: 生成选题清单

基于分析结果，生成10个选题：

```
1. [标题] - 预测互动率 X% - 理由：...
2. [标题] - 预测互动率 X% - 理由：...
...
```

### 选题分类模板

**优先级1（爆款潜力）**：免费/对比/真实体验
**优先级2（稳健型）**：教程/推荐/基础介绍  
**优先级3（测试型）**：趣味/脑洞/新角度

## 步骤5: 验证选题

对每个选题进行可行性检查：

- [ ] 是否有真实案例/数据支撑？
- [ ] 是否符合账号人设？
- [ ] 是否有差异化角度？
- [ ] 是否能引发评论互动？

```markdown
...

```

## 注意事项

1. **不要追所有热点** — 只追与账号定位相关的
2. **数据比直觉准** — 10条爆款的分析胜过100条猜测
3. **差异化是关键** — 同一个话题，找一个新角度
4. **时效性** — 热点24小时内发，干货长期有效
5. **降级准备** — 浏览器不可用时，用经验数据替代
